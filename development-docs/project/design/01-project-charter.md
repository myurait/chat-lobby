# Project Charter

## Purpose

ChatLobby は、複数の AI エージェント（Claude Code, Codex 等）を統合し、
スマホ起点の会話から調査・仕様化・実装依頼・修正依頼・状況確認までを一貫して行える
self-hosted 統合チャット基盤を構築するプロジェクトである。

## Problem Statement

現状、ChatGPT / Claude Code / Codex はそれぞれ分断されており、
ドキュメントの孤立、モデル選択の手動判断、ナレッジの分断、
スマホからの作業統制不足といった問題がある。
ユーザーは複数のツールを行き来しながら、同じ文脈を何度も再入力する必要がある。

## Goals

- スマホから自然言語で前面エージェントに指示し、背後のワーカーが自律的に作業を分担する
- 会話から仕様書を生成し、Git 正本として管理する
- Claude Code / Codex を統一インターフェースから起動・監視・制御する
- 仕様と実装の往復（差し戻しループ）を会話圏内で完結させる
- 他者が clone して試せる OSS として公開する

## Core Product Capabilities

- ユーザーは基本的に 1 つの前面エージェントと会話すればよく、調査相談、仕様相談、実装依頼、修正依頼を同じ会話圏で連続して行える。
- モデルやワーカーの選択は原則として基盤側が吸収し、必要時のみ手動 override を許可する。
- 主要文書には各エージェントが共通参照する Git 正本が存在し、会話で確定した内容を正本へ反映できる。
- 実装結果を受けて仕様書や決定記録を更新し、更新後の文書を次回以降の主要ナレッジとして再利用できる。
- 配下ワーカーの進行状況、作業中内容、完了状況、承認待ちを前面チャットから確認できる。
- プロジェクトごとの主要文脈は緩やかに分離しつつ、必要に応じて過去プロジェクトの規約・設計パターン・知見を横断参照できる。

## Important Secondary Capabilities

- スマホから現在の仕様書や関連文書を閲覧できる。
- 初期段階では閲覧優先としつつ、軽微な文書修正やコメント追記も将来的な対象に含める。
- 前面エージェントがローカルまたは共有リポジトリ上の最新コード状態を参照できる。
- 作業履歴と判断履歴を、最低限追跡可能な形で保存する。
- 会話中に得られた重要知見を、半自動または自動で共有ナレッジへ昇格できる余地を持つ。

## Non-Goals

- ChatGPT / Claude / Codex の 3 者すべてを対等に統合すること
- 完全自律で人間確認なしに全工程を無人化すること
- 厳密なプロジェクト管理ツールやチケット管理を主目的にすること
- 企業向けの厳格な権限制御や監査要件を初期段階から満たすこと

## Constraints

- Technical constraints
  - frontdoor は Open WebUI を採用し、本体改変は最小化する
  - Open WebUI は upstream を利用し、fork / vendoring はしない前提で進める
  - fork / vendoring が必要になった場合は、事前にユーザーの意思決定を仰ぐ
  - 文書の正本は Git に固定する
  - Open WebUI の plugin / tool / sidecar / adapter として拡張する
- Team constraints
  - 個人開発が前提
  - AI エージェントを積極的に開発に活用する
- Delivery constraints
  - 段階的にリリース（v0.1 ~ v1.0）
  - まず会話 → 文書 → 実装依頼の最短ループを成立させる
